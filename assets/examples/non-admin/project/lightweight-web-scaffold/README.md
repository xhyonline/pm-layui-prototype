# Lightweight Web Scaffold

这个目录是面向非后台页面的轻量 `layui-vue` 原型骨架。

它不是后台管理台，不包含左侧菜单、面包屑、tab 标签栏，也不默认套 `layui-vue-admin` 壳层。

## 设计目标

- 适合普通桌面 Web 原型，而不是后台系统
- 适合快速 `pnpm dev` 预览和自然语言迭代
- 结构中性，方便改造成搜索页、详情页、表单页、门户页、结果页
- 保持在 `layui-vue 2.23.3` 组件体系内

## 使用场景

- 用户要的不是后台，而是普通业务页面
- 需要一个可运行的轻量骨架，而不是只给单页代码
- 希望 AI 有一套默认的公共页结构可复用

## 使用方式

1. 复制本目录全部文件到目标工程目录。
2. 运行 `pnpm install`。
3. 运行 `pnpm dev`。
4. 基于现有 `views/` 替换或增删页面。
5. 如果页面超过 3 个，继续沿用 `src/router/` 和 `src/layouts/` 组织方式。

端口约束：

- 默认开发端口固定为 `5173`
- `vite.config.ts` 已启用 `strictPort: true`
- 如果 `5173` 已被占用，`pnpm dev` 应直接报错，而不是自动跳到 `5174+`

## 当前包含

- `PublicLayout` 公共页壳层
- 1 个场景首页：`/`
- 1 个搜索列表页：`/search`
- 1 个详情展示页：`/detail`
- 1 个信息提交页：`/form`
- 1 个结果反馈页：`/result`
- 1 组公共区块组件：`PageHero`、`SectionCard`
- 本地 mock 数据与中性样式变量

## 结构取向

这份骨架故意做成“页面型骨架”，而不是“固定站点模板”：

- 不预设营销站视觉
- 不写死行业术语
- 不把首页当作唯一核心页面
- 优先让搜索、详情、表单、结果这些高复用结构先具备

## 有意省略

- 登录体系
- 权限系统
- 真实 API
- 全局状态管理
- 国际化
- 交付级品牌视觉

它的目标是给非后台原型提供一个稳定起点，而不是替代正式官网或生产级站点工程。
